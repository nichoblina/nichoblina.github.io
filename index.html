Case study — kwek

Problem → build → result for the product I ship myself.

Solo product · Live at kwek.cards

Notes in. Study deck out.

Students waste hours rewriting notes into flashcards. kwek collapses that into a paste-or-upload flow that generates cards and quizzes with AI — then keeps everything usable offline.

Problem

Study tools that slow you down

Existing apps either ignore your own notes, or make you hand-author every card. AI tools dump raw output with no structure, no offline mode, and no trust that the content is actually quiz-ready.

Build

End-to-end, one person

Next.js 15 + TypeScript frontend, Supabase auth/storage/Postgres, Groq for fast generation with prompt guardrails, file preview with highlight mode, and offline-first sync so decks work without a connection.

Result

A real product, shipping

Live production app students can use today, plus a React Native companion in progress. Same designer/dev/ops loop — from UI to deploy — without a team.
